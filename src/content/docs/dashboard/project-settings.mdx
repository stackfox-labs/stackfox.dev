Every project has a **Settings** area reachable from the sidebar. Even though you open it from a specific project, it now includes three kinds of settings:

- organization settings
- project settings
- account settings

The members list and your role are organization-level, not project-level.

## Organization

At the top of the page you'll see an **Organization** section.

This section shows:

- the organization name
- your current role in that organization
- the organization members list

Owners and admins can rename the organization and manage members from here.

### Organization roles

Current roles in the dashboard include:

| Role | Can access projects | Can manage members | Can rename org | Can delete projects |
|---|---|---|---|---|
| `owner` | Yes | Yes | Yes | Yes |
| `admin` | Yes | Yes | Yes | Yes |
| `member` | Yes | No | No | No |

<Callout type="info" title="Organization-scoped access">
Members are added to the organization once, then gain access to that organization's projects through their org role. You do not manage separate member lists per project.
</Callout>

### Adding a member

Type a Roblox username in the `@username` field and click **Add**.

The user must have signed into StackFox at least once so their dashboard account exists.

Newly added members join with the `member` role.

### Removing a member

Owners and admins can remove non-owner members from the organization.

The organization owner cannot be removed from this screen.

## Project details

The **Project Details** card is still project-specific.

It lets you:

- rename the current project
- view the project ID
- see which organization the project belongs to
- see when it was created

Renaming a project only changes its display name. The project ID and API keys stay the same.

## API Keys

The Settings area links directly to the [API Keys page](/docs/getting-started/api-keys), where you can create, revoke, and rotate keys for the current project.

<Callout type="info" title="Revoking all keys">
The danger zone also includes **Revoke All API Keys**. This immediately invalidates all existing keys for the project and creates a fresh default key, so update your Roblox secret right away if you use it.
</Callout>

## Danger Zone

Project destructive actions still happen at the project level, even though access is controlled by your organization role.

### Revoke All API Keys

Invalidates every API key on the current project and generates a new default key.

Use this if you think a key was exposed.

### Delete Project

Deletes the current project and all of its project-scoped data, including:

- events
- records
- flows
- logs
- API keys

To confirm, type the exact project name and submit the dialog.

<Callout type="warning" title="Irreversible">
Project deletion is permanent. There is no recovery path after the project is deleted.
</Callout>
